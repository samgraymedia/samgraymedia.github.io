## Architecture Overview

The theme uses **Vite** as a build system to bundle JavaScript via ES6 modules. This provides:

- ✅ **Modern JavaScript** - ES6+ features (async/await, arrow functions, destructuring, etc.)
- ✅ **Code Splitting** - Organized into components, utilities, and modules
- ✅ **Bootstrap Integration** - Bootstrap JS components (modals, dropdowns, tooltips)
- ✅ **Development Server** - Instant feedback during development
- ✅ **Production Optimization** - Minified, tree-shaken output files

### Core Technologies

| Tool          | Purpose                         | Version |
| ------------- | ------------------------------- | ------- |
| **Vite**      | JavaScript bundler & dev server | ^5.0.0  |
| **Bootstrap** | UI framework with JS components | ^5.3.0  |
| **ESLint**    | Code linting and formatting     | ^8.53.0 |

## Setup & Installation

### 1. Install Dependencies

```bash
cd wp-content/themes/starter-theme-1.x/
npm install
```

This installs:

- `vite` - Build tool
- `bootstrap` - UI framework with JS components
- `sass` - CSS preprocessor
- `eslint` - Code linter
- `concurrently` - Run multiple npm scripts in parallel

### 2. Verify Installation

```bash

npm list

# Run a quick build test
npm run build
```

## Development Workflow

### Start Development Server

```bash
# Start Vite dev server + watch SASS
npm run watch

# Or run individually:
npm run dev          # Vite development server
npm run watch:css    # Watch SASS changes
```

**What happens:**

- Changes to `.js` files trigger Vite compilation
- Changes to `.scss` files trigger SASS compilation
- Assets are served from `http://localhost:3000` (HMR enabled)
- Your WordPress site still runs normally; scripts are bundled separately

### Code Changes During Development

1. **Edit a JS file** (e.g., `scripts/modules/search.js`)
2. **Vite compiles it instantly**
3. **Browser reflects change** (manually reload needed)
4. **Final bundle in `dist/site.js`** after you run `npm run build`

### Lint Your Code

```bash
# Check for code issues
npm run lint

# Auto-fix issues
npm run lint:fix
```

## Building for Production

### Build Once

```bash
# Compile both JS and CSS for production
npm run build

# This generates:
# - dist/site.js (minified JavaScript)
# - style.css (compiled CSS)
```

### What Gets Generated

After running `build`, `dist/site.js` contains:

- All your JavaScript code
- All imported utilities and modules
- Bootstrap JavaScript (minified)
- Vendor dependencies (minified)
- No comments or source maps

### Deploy

1. Run `npm run build` locally
2. Commit `dist/site.js` to version control
3. Deploy to production
4. WordPress automatically uses `dist/site.js` if it exists (via functions.php)

## Writing Modules

### Creating a New Module

Create a new file in `scripts/modules/` - e.g., `scripts/modules/gallery.js`:

```javascript
/**
 * Gallery Module
 * Handles image gallery interactions
 * @module modules/gallery
 */

import { querySelectorAll, addClass } from "../utils/dom.js";

/**
 * Initialize gallery functionality
 */
export function initGallery() {
  console.log("[Gallery] Initializing...");

  querySelectorAll("[data-gallery]").forEach((gallery) => {
    setupGallery(gallery);
  });
}

/**
 * Setup individual gallery
 */
function setupGallery(element) {
  element.addEventListener("click", (e) => {
    if (e.target.closest("[data-gallery-item]")) {
      handleGalleryClick(e);
    }
  });
}

/**
 * Handle gallery item click
 */
function handleGalleryClick(e) {
  const item = e.target.closest("[data-gallery-item]");
  // Your logic here
  console.log("Clicked gallery item:", item);
}

// Export for external use if needed
export { setupGallery };
```

### Import and Initialize in `index.js`

Edit `scripts/index.js`:

```javascript
// Add import at top
import { initGallery } from "./modules/gallery.js";

// Add to initTheme() function
function initTheme() {
  // ... other code ...
  initGallery(); // Initialize your module
}
```

## Using Utilities

### DOM Utilities (`utils/dom.js`)

```javascript
import {
  querySelector,
  querySelectorAll,
  addClass,
  removeClass,
  toggleClass,
  hasClass,
  show,
  hide,
  createElement,
} from "./utils/dom.js";

// Query elements
const button = querySelector(".my-button");
const items = querySelectorAll(".item");

// Manipulate classes
addClass(button, "active", "large");
removeClass(button, "disabled");
toggleClass(button, "highlight");

// Show/hide
show(element, "flex"); // Show with flex display
hide(element); // Hide (display: none)

// Create elements
const div = createElement(
  "div",
  { class: "card", id: "my-card" },
  "Card content",
);
```

### HTTP/AJAX Utilities (`utils/http.js`)

```javascript
import { ajax, get, post, put, del } from "./utils/http.js";

// WordPress AJAX (requires themeConfig.nonce from PHP)
const result = await ajax("my_action", {
  product_id: 123,
  quantity: 5,
});

// Standard fetch with error handling
const data = await get("/api/products");
const response = await post("/api/products", { name: "New Product" });
const updated = await put("/api/products/1", { name: "Updated" });
await del("/api/products/1");
```

### Event Utilities (`utils/events.js`)

```javascript
import {
  debounce,
  throttle,
  on,
  delegate,
  delay,
  once,
  observeElement,
} from "./utils/events.js";

// Debounce - wait until user stops typing
const handleSearch = debounce((e) => {
  console.log("Searching for:", e.target.value);
}, 300);
input.addEventListener("input", handleSearch);

// Throttle - limit scroll events
const handleScroll = throttle(() => {
  console.log("Scrolling...");
}, 250);
window.addEventListener("scroll", handleScroll);

// Event with cleanup
const cleanup = on(element, "click", () => {
  console.log("Clicked!");
});
// Later: cleanup(); // Removes listener

// Delegated events - for dynamic elements
delegate(document, ".dynamic-button", "click", (e) => {
  console.log("Dynamic button clicked");
});

// Lazy loading with IntersectionObserver
observeElement(".lazy-image", (entry) => {
  if (entry.isIntersecting) {
    entry.target.src = entry.target.dataset.src;
  }
});
```

## Creating Components

### Component Structure

Components wrap Bootstrap functionality with custom logic:

```javascript
/**
 * MyCustomModal Component
 * Extends Bootstrap Modal with custom behavior
 */

import { Modal } from "../components/Modal.js";
import { querySelector } from "../utils/dom.js";

export class MyCustomModal extends Modal {
  /**
   * Override onAfterOpen to add custom logic
   */
  onAfterOpen(event) {
    // Call parent
    super.onAfterOpen?.(event);

    // Custom logic
    console.log("Custom modal opened");
    this.loadFormData();
  }

  /**
   * Custom method
   */
  loadFormData() {
    const form = querySelector("form", this.element);
    // Load data...
  }
}
```

### Using in Twig

```twig
{# templates/components/my-form.twig #}
<div class="modal fade" id="myModal" data-js-component="modal">
    <div class="modal-dialog">
        <div class="modal-content">
            <div class="modal-header">
                <h5>My Form</h5>
                <button class="btn-close" data-bs-dismiss="modal"></button>
            </div>
            <div class="modal-body">
                <form>
                    <input type="text" placeholder="Name" />
                </form>
            </div>
        </div>
    </div>
</div>
```

### Auto-Initialization

Components with `data-js-component` attribute are auto-initialized in `index.js`:

```javascript
querySelectorAll('[data-js-component="modal"]').forEach((element) => {
  const modal = new Modal(element);
  element.__modal = modal; // Store for external access
});
```

Access from anywhere:

```javascript
const modal = document.getElementById("myModal").__modal;
modal.open();
```

## Common Patterns

### Pattern 1: Event Delegation for Dynamically Added Elements

```javascript
// Add handler to document; triggers for all matching elements (now + future)
delegate(document, ".add-to-cart", "click", async (e) => {
  const productId = e.target.dataset.productId;
  await ajax("add_to_cart", { product_id: productId });
});
```

### Pattern 2: Stop Repeated Initialization

```javascript
// Prevent duplicate module initialization
let initialized = false;

export function initSearch() {
  if (initialized) return;
  initialized = true;

  // Setup...
}
```

### Pattern 3: Module with Public API

```javascript
let state = { count: 0 };

export function initCart() {
  // Private setup
}

export function getCartCount() {
  return state.count;
}

export function updateCartCount(count) {
  state.count = count;
}
```

### Pattern 4: Cleanup on Page Navigation

```javascript
// For AJAX-loaded content, cleanup old listeners
export function cleanup() {
  // Remove event listeners
  document.querySelectorAll(".dynamic-element").forEach((el) => {
    el.removeEventListener("click", handler);
  });
}

// Called before loading new content
// window.addEventListener('pjax:start', cleanup);
```

### Pattern 5: Mobile Menu Toggle (Hamburger)

```javascript
/**
 * Hamburger Menu - Toggle navigation on mobile
 */
document.addEventListener("DOMContentLoaded", function () {
  const hamburger = document.querySelector(".hamburger");
  const navMenu = document.querySelector(".nav-menu");

  if (!hamburger || !navMenu) return;

  // Toggle menu on hamburger click
  hamburger.addEventListener("click", function () {
    hamburger.classList.toggle("is-active");
    navMenu.classList.toggle("is-active");
  });

  // Close menu when a link is clicked
  navMenu.querySelectorAll("a").forEach((link) => {
    link.addEventListener("click", function () {
      hamburger.classList.remove("is-active");
      navMenu.classList.remove("is-active");
    });
  });

  // Close menu when clicking outside
  document.addEventListener("click", function (event) {
    if (!event.target.closest("header")) {
      hamburger.classList.remove("is-active");
      navMenu.classList.remove("is-active");
    }
  });
});
```

**Files:** `scripts/hamburger.js`, `css/inc/global/nav.scss`

## Performance

### Code Splitting

Vite automatically optimizes:

- Dead code is removed (tree-shaking)
- Common dependencies deduplicated
- Vendor code separated if large

### Loading Optimization

Scripts load in footer (before `</body>`) via `wp_enqueue_script(..., true)`:

```php
wp_enqueue_script(
    'starter-theme-js',
    '/dist/site.js',
    array(...),
    '1.0.0',
    true  // Load in footer for performance
);
```

### Minimize Bundle Size

- Only import what you need
- Use tree-shaking friendly imports
- Lazy load heavy modules

```javascript
// ✅ Good - imports just what's needed
import { debounce } from "./utils/events.js";

// ❌ Avoid - imports entire module
import * as events from "./utils/events.js";
```

## Troubleshooting

### Issue: Changes not reflected in browser

**Solution:**

1. Rebuild: `npm run build`
2. Clear browser cache: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
3. Check console for errors: F12 > Console

### Issue: "themeConfig is undefined"

**Cause:** JavaScript loaded before PHP localization
**Solution:** `wp_localize_script()` called in `functions.php` - should be automatic

### Issue: Bootstrap components not working

**Causes:**

- Bootstrap JS not enqueued (check in DevTools Network tab)
- Element not in DOM when script runs
- jQuery conflict

**Solution:**

- Verify `bootstrap-js` loads before `starter-theme-js`
- Use `waitForElement()` utility for dynamically added elements

```javascript
import { waitForElement } from "./utils/events.js";

const modal = await waitForElement("#myModal", 5000); // Wait 5 seconds
new bootstrap.Modal(modal).show();
```

### Issue: ESLint errors

```bash
# Auto-fix issues
npm run lint:fix

# For specific rules, edit .eslintrc.json
```

### Issue: Module not found error

**Check:**

1. File path is correct relative to importing file
2. Using `.js` extension in import: `import { x } from './file.js'`
3. Exported function name matches: `export function X` vs `import { X }`
